Контентные изображения

Для добавления изображения на страницу используется тег <img> .

2 основных атрибута:

  • src - задает путь до изображения;
  • alt - задает альтернативный текст для изображения (отобразится, если изображение не загрузилось, а также читается скринридером). 
<img src="image.jpg" alt="Это картинка">

Форматы изображений в вебе

  • JPEG - хорош для фотографий, плох для текста, графики. Не поддерживает прозрачность;
  • PNG - хорош для схем, чертежей, текста, графики, фотографий (обычно проигрывает JPEG по размеру файла). Поддерживает полупрозрачность;
  • GIF - используется только для анимации;
  • SVG - масштабируемая векторная графика. Хорош для иконок, мелкой графики (логотипы, иллюстрации). Плюсы: масштабируемость, возможность стилизации через CSS, анимации через Javascript, малый размер файла. Поддержка браузерами: IE9+.

SVG. Способы добавления векторной графики на страницу

1. Через тег <img> как обычное изображение (поддержка IE9+):

<img src="image.svg">

2. Через свойство background-image (поддержка IE9+):

.block {
    width: 100px;
    height: 50px;
    background: url("image.svg");
    background-size: 100px 50px;
}

Обратите внимание, что для селектора .block задан размер background-size. Это необходимо, если исходный файл SVG имеет размер больше блока, иначе будет видна только верхняя левая часть изображения SVG.

Проблема способов 1 и 2: нет возможности управлять свойствами SVG-изображения через CSS.

SVG. Способы добавления векторной графики на страницу

3. Добавление SVG непосредственно в HTML-документ (поддержка IE9+):

<svg>
    <rect width="100" height="100"
      stroke-width="1" stroke="rgb(0,0,0)"/>
</svg>

Любое сложное SVG изображение (например, созданное в Adobe Illustrator или найденное в интернете) можно открыть в текстовом редакторе, скопировать его код и вставить непосредственно в HTML.

Этот приём может быть полезным, так как изображение встроено прямо в документ, и для его загрузки не происходит дополнительный HTTP-запрос.

Недостатки: вероятность получения очень тяжелого документа, наличие блоков SVG-кода в нем и невозможность кэширования.

Тегам внутри тега <svg> можно задавать классы и прописывать этим классам css-свойства, например, менять цвет изображения.

4. Через тег <object> (поддержка IE9+):

<object type="image/svg+xml"
  data="image.svg"></object>

SVG. Оптимизация

В коде SVG-файлов содержится много служебной информации, которая никак не влияет на отображение SVG, но увеличивает вес файлов. Для оптимизации SVG-файлов можно воспользоваться онлайн-ресурсами6 или плагином для инструментов автоматизации (Gulp, Grunt, Webpack).

Оптимизация изображений

Современные браузеры позволяются делать изображения по-настоящему отзывчивыми. Для этого применяются тег <picture> и 2 атрибута тега <img> : srcset и sizes .

Тег <picture> и атрибуты srcset и sizes позволяют браузеру на этапе парсинга HTML выбрать подходящее изображение, в зависимости от ширины окна браузера (viewport) и плотности пикселей экрана (device pixel ratio).

В результате браузер не будет скачивать огромное изображение, предназначенное для десктопов с высокой четкостью, если вы зайдете на сайт с телефона. Профит: быстрая загрузка изображений на устройствах с небольшой шириной экрана и, как правило, ограниченной скоростью интернета.

Браузерная поддержка атрибутов srcset и sizes : Edge13+, Firefox, Chrome, Opera, Safari9+, iOS Safari9.2+, Android5+.

Браузерная поддержка тега <picture> : Edge13+, Firefox, Chrome, Opera, Safari9.1+, iOS Safari9.3+, Android5+.

Есть полифил Picturefill для неподдерживающих браузеров13.

Атрибут srcset

Атрибут srcset позволяет определить наиболее подходящее изображение для конкретного экрана (в зависимости от его пиксельной плотности или ширины).

Пример 1. Разный файл с изображением для разной пиксельной плотности экрана (dpi).

На стандартных экранах загрузится файл wombat_1x.jpg, на экранах с повышенной пиксельной плотностью (например, Retina) загрузится файл wombat_2x.jpg.

Атрибут sizes

При парсинге html и загрузке изображений браузер не знает о размерах изображений, заданных в css (css обрабатывается после html). Он предполагает, что изображение будет занимать всю ширину вьюпорта (так работает атрибут srcset ).

Что делать, если изображение занимает не всю ширину вьюпорта?

Для того чтобы браузер при загрузке страницы мог выбрать подходящее изображение в зависимости от его размера, в дополнение к атрибуту srcset используется атрибут sizes .

Атрибут sizes не изменяет размеры изображения (это делается в css), он просто указывает браузеру, какой ширины будет изображение при конкретной ширине вьюпорта (соответствует медиа-запросам в css).

Пример. Разный файл с изображением для разного размера экрана и разного размера изображения.

На экранах шириной менее 400px загрузится файл wombat_400.jpg, на экранах от 401 до 640px загрузится файл wombat_640.jpg, на экранах от 641 до 1280px также загрузится файл wombat_640.jpg (так как с ширины экрана 640px изображение занимает только 50% экрана), и только при ширине экрана более 1280px загрузится файл wombat_1024.jpg.

Тег picture

Тег <picture> так же, как и атрибут srcset указывает браузеру, какой файл с изображением выбрать для определенных параметров экрана.

Помимо этого, в теге <picture> можно указать несколько файлов с разным форматом (например, изображение в современном, но плохо поддерживаемом формате webp, и изображение в формате jpeg для браузеров без поддержки webp).

Еще одна возможность, которую дает тег <picture> , - это подключение разных изображений или частей изображения в зависимости от ширины вьюпорта.

Пример. Разные изображения для разного размера экрана.

На ширине до 1000px подключится файл wombat_small.jpg, в котором вомбат занимает центральное место. На ширине свыше 1000px подключится файл wombat_large.jpg, в котором в основном изображена природа, и вомбат не занимает центральное положение.

Иконки

Способы создания иконок:

  • подключение картинок через тег img. Плохо, так как каждая картинка - отдельный запрос на сервер!
  • PNG-спрайты. Вымерли;
  • иконочные шрифты. Вымирают;
  • векторная графика (SVG), SVG-спрайты;
  • иконки на чистом CSS.

Иконочные шрифты

Иконочный шрифт - шрифт, каждый символ которого представлен иконкой.

Преимущества:

  • масштабируемость;
  • возможность применения CSS-свойств;
  • 1 HTTP-запрос для всех иконок;
  • маленький размер файла.

Недостатки:

  • иконочный шрифт может не загрузиться;
  • монохромность;
  • иконочный шрифт - хак, несемантичное использование;
  • возможно неодинаковое отображение в разных браузерах.

Использование иконочного шрифта (fontello.com)

Шаг 1. Выбираем/загружаем нужные иконки и нажимаем на кнопку Download webfont.

Использование иконочного шрифта (fontello.com)

Шаг 2. В скачанной папке заходим в папку fonts и копируем из нее все файлы со сгенерированными шрифтами в проектную папку fonts.

Использование иконочного шрифта (fontello.com)

Шаг 3. Подключаем иконочный шрифт в CSS (копируем целиком сгенерированный файл со стилями в наш стилевой файл).

Основное в этом файле:

1. Подключение иконочного шрифта:

@font-face {
    font-family: 'my-icon-font';
    src: url('../font/my-icon-font.eot?32949143');
         /* подключение всех типов шрифтов */
    font-weight: normal;
    font-style: normal;
}

2. Стили для всех классов, начинающихся с "icon-":

[class^="icon-"]:before, [class*=" icon-"]:before {
    font-family: "my-icon-font";
    font-style: normal;
    font-weight: normal;
}

3. Названия всех иконок, входящих в данный шрифт:

.icon-emo-happy:before { content: '\e800'; }
.icon-emo-wink:before { content: '\e801'; }
.icon-emo-wink2:before { content: '\e802'; }
.icon-emo-unhappy:before { content: '\e803'; }

Использование иконочного шрифта (fontello.com)

Шаг 4. В html-разметке в нужном месте вставляем тег <i> с нужным классом (названием нужной иконки).

<p>
    <i class="icon-emo-happy"></i>
        Тестируем иконочный шрифт
    <i class="icon-emo-wink2"></i>
</p>

Добавим стилей:

p i {
    font-size: 3rem;
    color: red;
}

Поддержка иконочного шрифта

Для добавления новых иконок в существующий иконочный шрифт можно воспользоваться файлом config.json, который автоматически создается при генерации иконочного шрифта на fontello.com.

Все, что нужно сделать, это загрузить файл config.json на fontello.com методом drag-and-drop (самый верхний раздел Custom Icons), и все иконки из шрифта отобразятся на сайте.

После этого останется добавить новые иконки и повторить шаги 1-4.

SVG-спрайты

Назначение SVG-спрайта: упростить и структурировать работу с мелкой графикой (иконками).

Основные способы создания SVG-спрайтов:

  • SVG-спрайт из CSS файла;
  • SVG-спрайт с общим viewBox;
  • SVG-спрайт инлайново в HTML.

Подробнее про использование первых двух способов см. здесь11.

SVG-спрайт из CSS-файла

Преимущества:

  • иконки кэшируются браузером, так как кэшируется css-файл.

Недостатки:

  • иконки нельзя стилизовать через css;
  • не поддерживается анимация.

SVG-спрайт с общим viewBox

Преимущества:

  • кэширование изображений;
  • можно отображать svg-картинки разными способами: через тег svg, img, background-image.

Недостатки:

  • частичная браузерная поддержка (не поддерживают Safari, Android Browser).

SVG-спрайт инлайново в HTML

Шаг 1. В HTML вставляем контейнер для спрайта:

<svg style="display:none;">
    <!-- здесь будут иконки -->
</svg>

Шаг 2. В контейнер спрайта вставляем код каждой иконки в тег-обертку <symbol> :

<svg style="display:none;">
    <symbol id="icon-facebook" viewBox="0 0 49.652 49.652">
        <path d="M24.826,0C11.137 …"></path>
    </symbol>
</svg>
Атрибут viewBox тега <symbol> имеет 4 значения. Первые два (как правило, 0, 0) задают координаты левого верхнего угла изображения. Третье и четвертое значения определяют размер изображения.

Шаг 3. Обращаемся к нужной иконке в нужном месте страницы при помощи тега <use> :

<svg class="icon">
    <use xlink:href="#icon-facebook"></use>
<svg>

SVG-спрайт инлайново в HTML. Преимущества и недостатки

Преимущества:

  • прост в понимании;
  • автоматическая генерация спрайтов (при помощи инструментов автоматизации).

Недостаток:

  • SVG-спрайт не кэшируется.

Чтобы исправить этот недостаток, можно ссылаться на внешний SVG-спрайт вместо включения его в документ инлайново:

<svg class="icon">
    <use xlink:href="path/to/icons.svg#icon-facebook"></use>
<svg>

Недостаток: не поддерживается IE.

Решение: плагин svg4everybody12.

Поясняющий текст для иконок в HTML

Для доступности необходимо указывать поясняющий текст для иконок (аналог атрибута alt для изображений), который будет визуально скрыт, но будет доступен для скринридеров.

Способ 1. Класс-хелпер .text-hide

<span class="icon">Facebook</span>
Facebook
<span class="icon text-hide">Facebook</span>
.text-hide {
    text-indent: 100%;
    white-space: nowrap;
    overflow: hidden;
}
Facebook

Поясняющий текст для иконок в HTML

Способ 2. Класс-хелпер .visually-hidden (более универсальный, подходит не только для текста)

.visually-hidden {
    position: absolute !important;
    clip: rect(1px 1px 1px 1px); /* IE6, IE7 */
    clip: rect(1px, 1px, 1px, 1px);
    padding:0 !important;
    border:0 !important;
    height: 1px !important;
    width: 1px !important;
    overflow: hidden;
}

Поясняющий текст для иконок в HTML

Способ 3. Указание поясняющего текста в атрибуте title (только при использовании инлайнового <svg> )

<svg width="100" height="100" viewBox="0 0 100 100"
    aria-labelledby="title desc">
    <title id="title">Title</title>
    <desc id="desc">Description</desc>
    <text y="-10">Text</text>
    <path d="..."/>
</svg>
