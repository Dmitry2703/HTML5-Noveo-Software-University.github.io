Боксовая модель

Box-sizing (content-box vs border-box)

По умолчанию используется значение content-box. Это означает, что width и height не включают в себя внутренние отступы, рамки и внешние отступы.

<div class="block block--default">Content-box</div>
<div class="block block--border-box">Border-box</div>
.block {
    background: #c5c5c5;
    border: 5px solid black;
    margin: 10px;
    padding: 20px;
}

.block--default {
    width: 150px;
    height: 150px;
}

.block--border-box {
    width: 200px;
    height: 200px;
    box-sizing: border-box;
}

Блочные и строчные элементы

В чем разница между строчными и блочными тегами?

В чем разница между строчными и блочными тегами?

  • Расположение -
    блочный элемент - всегда с новой строки. Строчный - справа от предыдущего строчного, если достаточно места.
  • Боксовая модель -
    блочный элемент по умолчанию занимает 100% ширины, строчный - только необходимое ему пространство. Вертикальные паддинги игнорируются при расчете высоты строчного элемента
  • Пробелы между блоками -
    если между закрывающим и следующим открывающих тегами двух строчных элементов есть пробел, табуляция или перенос строки, то браузер отобразит это как пробел между словами.
  • Margin -
    к строчным элементам применяется только горизонтальный margin

Margin Collapse

Margin collapse с родительским блоком

Избавиться от схлапывания margin'ов дочернего элемента с родительским блоком можно с помощью:

  • Задания родительскому блоку свойства padding
  • Задания родительскому блоку свойства overflow со значением: auto, scroll или hidden
  • Применения к родительскому блоку зачистки (clearfix)

Поток

Если к элементам на странице не применять никаких стилей, то они будут находиться в нормальном потоке. Высота родительского элемента будет включать в себя сумму высот дочерних элементов.

Элемент выбивается из потока, если к нему применяется:

  • float
  • Отрицательный вертикальный margin (у блочных элементов)
  • position: absolute; или position: fixed;
  • display: none;

Влияние трансформации родительского блока на его позиционирование и позиционирование дочерних блоков

Блок, к которому применяется transform, создает новый контекст наложения, ведет себя так, как если к нему применить position: relative . Поэтому дочерние элементы с position: absolute будут позиционироваться относительно этого родительского элемента с трансформацией.

Фиксированные ( position: fixed ) элементы внутри блока, к которому применяется transform, будут вести себя как элементы с position: absolute. Т.е. они будут позиционироваться относительно родитеского элемента с трансформацией, их можно "проскроллить", будто они не фиксированные.
