Что такое фреймворки и зачем они нужны

Фреймворк — каркас, структура.

Фреймворк — каркас, структура. Это программная платформа, определяющая структуру программной системы; программное обеспечение, облегчающее разработку и объединение разных компонентов большого программного проекта.

CSS-фреймворк — фреймворк, созданный для упрощения работы верстальщика, быстроты разработки и исключения максимально возможного числа ошибок вёрстки (проблемы совместимости различных версий браузеров и т. д.). Как и библиотеки скриптовых языков программирования, CSS-библиотеки, обычно имеющие вид внешнего css-файла, «подключаются» к проекту (добавляются в заголовок веб-страницы).

CSS фреймворки:

  • Bootstrap;
  • Foundation;
  • Maxmertkit;
  • ...

Плюсы:

  • более быстрая разработка;
  • кроссбраузерность;
  • легкая настраиваемость;
  • единообразие кода при работе в команде.

Минусы:

  • может быть много лишнего кода, не используемого в конкретном проекте;
  • при постоянном использовании фреймворков разработчик может превратиться в "настройщика фреймворков" и "интегратора плагинов".

Bootstrap

На данный момент, Bootstrap (продукт компании Twitter) - самый популярный фреймворк для фронтенда.

3 основных составляющих:

  • css компоненты;
  • javascript компоненты (требуется подключение библиотеки jQuery);
  • иконочный шрифт (250+ иконок).

Подключение Bootstrap

Способ 1. Подключение готовых файлов, скачанных с сайта Bootstrap

Для использования bootstrap подключаем его в теге <head> :

<link rel="stylesheet" href="path-to-file/bootstrap.min.css">

Если используются js-компоненты, то подключаем jQuery и bootstrap:

<script src="path-to-file/jquery.min.js"></script>
<script src="path-to-file/bootstrap.min.js"></script>

Подключение Bootstrap

Способ 2. Подключение готовых файлов через CDN

<link rel="stylesheet" href="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.7/css/bootstrap.min.css">
<script src="https://code.jquery.com/jquery-3.1.1.min.js"></script>
<script src="https://maxcdn.bootstrapcdn.com/bootstrap/3.3.7/js/bootstrap.min.js"></script>

Подключение Bootstrap

Способ 3. Использование исходных файлов (написанных на Less или Sass)

Алгоритм:

1. Скачать bootstrap с сайта вручную или при помощи npm.

2. Подключить bootstrap в главном .scss файле (например, index.scss) при помощи директивы @import :

@import "../../node_modules/bootstrap-sass/assets/stylesheets/_bootstrap.scss";

3. Скомпилировать .scss файлы в css при помощи сборщика (gulp, grunt, webpack).

Кастомизация Bootstrap

Для чего настраивать (или кастомизировать) Bootstrap?

Причин две: во-первых, скорее всего вам понадобятся не все компоненты Bootstrap, а во-вторых, вес такого шаблона даже в сжатом виде немал — 530 Кб. Добавьте 90 Кб jQuery, без которого js-компоненты работать не будут. Поэтому все лишнее следует удалять.

Способ 1. Настройка вручную на сайте Bootstrap

http://getbootstrap.com/customize/

1. Выбрать необходимые компоненты (как правило, все Common CSS и нужные из Components и JavaScript components).

2. В блоке Less Variables задать необходимые значения любых переменных Bootstrap.

3. Скачать скомпилированные файлы.

Кастомизация Bootstrap

Способ 2. Настройка в проектной папке

1. Скачать исходные файлы Bootstrap (.less или .scss).

2. Создать файл, например, bootstrap-custom.scss в папке со стилями проекта и подключить только необходимые компоненты из Bootstrap при помощи директивы @import :

@import "bootstrap/dropdowns";
@import "bootstrap/navbar";
@import "bootstrap/breadcrumbs";

3. Подключить bootstrap-custom.scss в главном .scss файле проекта.

4. При необходимости переопределить переменные Bootstrap в проектном файле, например, в vars.scss:

$gray-base: #ccc; // по дефолту #000

5. При необходимости переопределить стили для Bootstrap классов в проектных файлах (они должны поключаться после файла c Bootstrap).

Сеточная система

Особенности:

  • 12 колонок;
  • responsive;
  • mobile first.

Основные правила построения сетки:

  • строки помещаются в блок .container;
  • строки с классом .row формируют горизонтальные группы столбцов;
  • контент помещается в блоки-колонки с классами .col-*.

Сеточная система

Классы для создания колонок

1. Каждый класс .col-* имеет в названии префикс, указывающий, на какой ширине экрана будет применяться данный класс:

Screen sizeExtra small devices Phones (<768px)Small devices Tablets (≥768px)Medium devices Desktops (≥992px)Large devices Desktops (≥1200px)
Class prefix .col-xs-.col-sm- .col-md- .col-lg-

Сеточная система

Классы для создания колонок

2. Каждый класс .col-* имеет 2 свойства:

.col-* {
    float: left;
    width: x;
}

Ширина колонки = 100% * n / 12 ,

где n - число в названии класса .col-* .

.col-xs-1 :

ширина = 100% * 1 / 12 = 8,3%

.col-xs-12 :

ширина = 100% * 12 / 12 = 100%

Сеточная система

Смещение колонок

Сдвиг колонок осуществляется при помощи классов .col-*-offset-* .

Каждый класс .col-*-offset-* имеет свойство:

.col-*-offset-* {
    margin-left: x;
}

Порядок колонок

Порядок колонок легко изменить при помощи классов .col-*-push-* и .col-*-pull-* .

Каждый класс .col-*-push-* и .col-*-pull-* имеет свойство left или right .

Модальные окна

Основные элементы для создания модального окна

Кнопка:

  • data-toggle="modal" - показывает модальное окно;
  • data-target="#myModal" - указывает id окна, которое нужно открыть.

Модальное окно:

  • .modal - определяет содержимое окна;
  • .fade - добавляет эффект плавного появления/затухания окна;
  • tabindex="-1" - закрытие окна по нажатию клавиши Esc;
  • .modal-dialog - устанавливает ширину и отступы у окна;
  • .modal-content - добавляет стили Bootstrap к окну.

Табы

Основные элементы для создания табов:

  • .nav, .nav-tabs - добавляет стили Bootstrap для создания табов;
  • data-toggle="tab" - активизирует переключение табов;
  • href="#tab1" - указывает id блока, который нужно отобразить;
  • переключение контента (блок с классом .tab-pane) в блоке-обертке .tab-content осуществляется за счет переключения класса .active у блоков .tab-pane при клике на соответствующий таб.
